# Swarm Drone Search & Rescue

A swarm of autonomous drone agents explores a 20x20 grid-based disaster site to locate survivors.

The environment is dynamic: fire spreads probabilistically each timestep, agents have partial observability (a sensing radius of 2), and agents can be destroyed by fire. Three coordination strategies are compared across 810 controlled simulation runs.

This project was developed as part of the COMP3004 Designing Intelligent Agents module at the University of Nottingham.

## Strategies

- **Random**: reactive random walk (baseline)
- **A* Planned**: deliberative frontier-seeking with A* pathfinding
- **Pheromone**: Ant Colony Optimisation-inspired stigmergic coordination

## Installation

Clone the repository:

```bash
git clone https://github.com/rubenodamo/swarm-drone-search-rescue.git
cd swarm-drone-search-rescue

# Install dependencies
pip install -r requirements.txt
```

## Running Experiments

Run all 810 simulations (3 strategies x 3 swarm sizes x 3 hazard rates x 30 seeds):

```bash
python experiments/run_experiments.py
```

Results are written to `results/raw/` as one CSV per run.

Run timeseries data collection (90 runs at swarm_size=6, hazard_rate=medium):

```bash
python experiments/run_timeseries.py
```

Run coverage heatmap data collection (same 90 runs):

```bash
python experiments/run_heatmaps.py
```

## Generating Figures and Statistics

```bash
python experiments/analyse_results.py
```

This produces:
- `results/all_runs.csv`: flat table of all 810 runs
- `results/summary.csv`: mean and std per condition
- `results/significance_tests.txt`: Kruskal-Wallis and Mann-Whitney U pairwise tests
- `results/figures/`: all 7 report figures

## Live Visualisation

```bash
python visualisation/playground.py
```

Opens an animated Tkinter window with the 20x20 grid. Use the left panel to select strategy, swarm size, hazard rate, and seed, then press Play. Supports pause, single-step, and reset.

## Running Tests

Run the test suite using pytest:

```bash
pytest
```

Optional:

```bash
# Run with verbose output
pytest -v
```

## Project Structure

```bash
├── src/
│   ├── environment/            # DisasterGrid, obstacle, survivor, and fire logic
│   ├── agents/                 # DroneAgent base, RandomDrone, AStarDrone, PheromoneDrone
│   ├── model/                  # DisasterModel, shared pheromone grid
│   ├── strategies/             # astar.py
│   └── metrics/                # MetricsCollector, CSV writer
│
├── experiments/                # Experiment execution logic 
│   ├── run_experiments.py      
│   ├── run_timeseries.py       
│   ├── run_heatmaps.py         
│   └── analyse_results.py      
│
├── results/
│   ├── raw/                    # One CSV per run (gitignored)
│   └── figures/                # PNG charts generated by analyse_results.py
│
├── tests/                      # Unit tests
│
└── visualisation/
    └── playground.py           # Tkinter animated playground
```
